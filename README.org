=kubed-ext= is an extension pack for [[https://github.com/eshelyaron/kubed][kubed]].  It keeps kubed as the base Kubernetes
client and adds the day-to-day comforts people expect from tools like k9s:
better columns, async top views, port-forward management, Strimzi actions,
CRD columns, filtering, marking, terminals, and quieter error handling.

No extra Emacs async framework is required.  The package uses kubed, transient,
and native Emacs processes.

* Highlights

- *k9s-style resource lists*: richer columns for pods, nodes, services,
  ingresses, PVs/PVCs, workloads, events, autoscalers, PDBs, and CRDs.
- *CRD auto-discovery*: reads =additionalPrinterColumns= from CRDs and injects
  them into kubed list buffers.  Strimzi Kafka views include k9s-like metadata
  state and quieter warning display.  CRD metadata is cached on disk so resource
  switching has custom resources immediately after startup.
- *Async top*: non-blocking pod/node metrics buffers backed by =kubectl top=.
- *Port-forwards*: discover ports from pods, services, and deployments; manage
  active forwards from one buffer; highlight rows with active forwards.
- *Filtering*: server-side label selectors plus substring filtering that dims
  non-matches and lets you jump between hits.
- *Marking*: Dired-like marks for batch delete and batch log workflows.
- *Strimzi helpers*: pause/resume reconciliation, restart connectors, scale
  KafkaConnect, and adjust KafkaTopic partitions.
- *Pod terminals*: open vterm, eat, ghostel, eshell, ansi-term, or one-shot
  shell commands into pods.
- *Rollouts*: history, revision inspection, undo, and restart/jab actions.
- *Less noisy refreshes*: routine refreshes avoid spamming =*Messages*=; errors
  show where they help.

* Quick start

1. Run =M-x kubed-list-pods=.
2. Press =?= for the context-aware menu.
3. Press =C= to change context or =N= to change namespace.
4. Press =R= to switch resource type.
5. Press =g= to refresh.

Useful first commands:

| Key / command | What it does |
|---------------+--------------|
| =?= | Open kubed-ext transient menu |
| =d= | Describe resource at point |
| =W= | Open =kubectl get -o wide= view |
| =S= | Set server-side label selector |
| =f= | Set substring filter |
| =M-n= / =M-p= | Jump filter matches |
| =m= / =u= | Mark / unmark row |
| =M-m= / =M-u= | Mark / unmark all visible rows |
| =X= | Delete marked resources |
| =c= | Copy menu |

* Common workflows

** Metrics

| Key / command | Scope |
|---------------+-------|
| =M-x kubed-ext-top-pods= | pod metrics |
| =M-x kubed-ext-top-nodes= | node metrics |
| =T= or =M= in pod list | pod metrics |
| =M= in node list | node metrics |
| =T= on a node | single-node top summary |

** Port-forward

| Resource | Key |
|----------+-----|
| Pod | =F= |
| Service | =F= |
| Deployment | =F= |
| Management buffer | =M-x kubed-ext-list-port-forwards= |

Port candidates come from live pod/service/deployment specs.  Active forwards
are tracked and highlighted in list buffers.

** Logs and terminals

| Key / command | What it does |
|---------------+--------------|
| =i= on pod | initContainer logs |

| =t= on pod | eat into pod |
| =G= on pod | ghostel into pod |
| =E= on pod | eshell via kubed-tramp |
| =A= on pod | ansi-term |
| =&= on pod | prompt for one shell command |

|-----+---------|

* Configuration

#+begin_src emacs-lisp
;; Shell used by pod terminal commands.
(setq kubed-ext-pod-shell "/bin/bash")

;; Display resource buffers as YAML or JSON.
(setq kubed-ext-output-format "json")

;; Use a non-default kubectl path.
(setq kubed-kubectl-program "/usr/local/bin/kubectl")

;; Customize status colors.
(setf (alist-get "Running" kubed-ext-status-faces nil nil #'string=) 'bold)
(push '("Degraded" . error) kubed-ext-status-faces)

;; Enable command log when you want audit/debug detail.
(setq kubed-ext-command-log-enabled t)

;; CRD metadata cache.  Default:
;; ~/.emacs.d/.cache/kubed-ext/crds/
(setq kubed-ext-crd-cache-enabled t)
(setq kubed-ext-crd-cache-directory
      (expand-file-name ".cache/kubed-ext/crds/" user-emacs-directory))
#+end_src

* Notes

- CRD discovery is cached in memory and on disk by context.  Cached CRDs load
  first, then kubed-ext refreshes from the cluster asynchronously and rewrites
  the cache when CRD metadata changes.
- Use =M-x kubed-ext-refresh-crd-columns= when you want an immediate manual CRD
  refresh.  It does not delete the disk cache.
- Label selectors are sent to kubectl as =--selector=, so large lists filter on
  the API server.
- Substring filtering is visual only: non-matching rows are dimmed, not removed.
- Routine refreshes are intentionally quiet.  Look at error banners, top buffer
  headers, or the command log when debugging.

* Troubleshooting

** Metrics are empty

Check metrics-server first:

#+begin_src bash
kubectl top nodes
kubectl top pods -A
#+end_src

** CRD columns are missing

Run =M-x kubed-ext-refresh-crd-columns= and reopen the list buffer.  If a CRD
does not define =additionalPrinterColumns=, kubed-ext has little to display
beyond name/age unless the resource has a built-in extension.

** Port-forward fails

Check the resource, namespace, context, and local port:

#+begin_src bash
kubectl get <type> <name> -n <namespace>
lsof -i :<local-port>
#+end_src

** Pod terminal does not open

Install the backend you are using (=vterm=, =eat=, or =ghostel=), or use
built-in =ansi-term= / shell command support.  Also verify the pod has
=/bin/sh= or your configured =kubed-ext-pod-shell=.

** Too much command output

Keep =kubed-ext-command-log-enabled= nil for normal use.  Enable it only while
debugging kubectl calls.

* Development

- Keep public names under =kubed-ext-= and internal names under =kubed-ext--=.
- Preserve kubed behavior unless the extension is deliberately adding a feature.
- Prefer async =make-process= for cluster calls that can block.
- Handle =nil=, empty string, =<none>=, and =<unknown>= consistently.
- Byte-compile cleanly before submitting changes.

#+begin_src bash
emacs -Q --batch \
  --eval "(package-initialize)" \
  -L /path/to/kubed \
  -L . \
  --eval '(byte-compile-file "kubed-ext.el")'
#+end_src
